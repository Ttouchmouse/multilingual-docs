# MVP_SCOPE: 화면 기반 다국어 텍스트 QA 맵

## 1. MVP 목표

MVP는 피그마 기준 주요 화면 이미지와 기존 HTML 번역 데이터를 연결하여, 화면 맥락 안에서 11개 언어 번역본을 조회·검수할 수 있는 개인용 도구를 만드는 것을 목표로 한다.

핵심은 자동화가 아니라, 다음 구조를 빠르게 구현하는 것이다.

`화면 이미지` → `텍스트 영역 지정` → `번역 key 연결` → `11개 언어 기본 노출`

## 2. 포함 범위

## View Mode 수정 원칙

화면 추가, 이미지 업로드, 텍스트 영역 지정, 번역 key 연결 기능은 MVP에 유지한다.

단, 기본 진입 화면은 등록/편집 UI가 아니라 조회 중심 View Mode로 구성한다.

등록된 화면이 존재하는 경우:
- 좌측에는 화면 리스트를 표시한다.
- 중앙에는 선택된 화면 이미지를 표시한다.
- 화면 위에는 연결된 텍스트 영역 overlay를 표시한다.
- 우측에는 해당 화면의 다국어 번역 테이블을 표시한다.

화면 추가/편집 기능은 삭제하지 않는다. 다만 기본 조회 화면의 주 UI로 노출하지 않고, 별도 버튼 또는 모드 전환을 통해 접근한다.

등록된 화면이 없는 경우에만 Empty State 또는 화면 추가 유도 UI를 표시한다.
등록된 화면이 하나 이상 존재하는 경우에는 반드시 View Mode를 먼저 표시한다.

### 2.1 번역 데이터
- HTML 파일 업로드
- HTML 내 번역 테이블 파싱
- 11개 언어 번역 데이터 저장
- key, KR, EN 기준 검색
- 전체 언어 컬럼 표시

### 2.2 화면 관리
- 화면 이미지 업로드
- 화면명 입력
- 화면 그룹 선택
- 플랫폼 선택
- 기준 언어 설정
- Figma URL 입력
- 화면 메모 입력

### 2.3 텍스트 영역 관리
- 화면 이미지 위에 텍스트 영역 수동 생성
- 텍스트 영역 이동
- 텍스트 영역 크기 조정
- 텍스트 영역 삭제
- 화면 표시 문구 입력

### 2.4 번역 항목 연결
- 텍스트 영역에 번역 항목 연결
- KR, EN, key로 번역 항목 검색
- 검색 결과 중 사용자가 직접 선택
- 연결 해제
- 연결 상태 설정

### 2.5 화면 기준 번역 리스트
- 오른쪽에 화면 내 텍스트별 11개 언어 번역본 기본 노출
- 각 항목에 key, 화면 문구, 상태, 메모 표시
- 화면 텍스트 클릭 시 오른쪽 항목 하이라이트
- 오른쪽 항목 클릭 시 화면 텍스트 영역 하이라이트
- 선택 항목 자동 스크롤

### 2.6 상태 및 메모
- 미연결
- 연결 완료
- 확인 필요
- 번역 누락
- 수정 필요
- 텍스트 영역별 메모 저장

## 4. 우선 적용 화면

MVP에서 우선 등록할 화면은 문구 검수 가치가 높고, 텍스트 수가 과도하게 많지 않은 화면으로 제한한다.

권장 우선순위:

1. 결제/구독 화면
2. 성인인증 화면
3. 쿠키 동의 팝업
4. 회원가입/로그인
5. 뷰어 경고문/알럿
6. 이벤트/프로모션 팝업
7. 마이페이지/계정 설정

## 5. MVP 완료 기준

MVP는 다음 조건을 만족하면 완료로 본다.

- 번역 HTML 파일을 업로드하고 파싱할 수 있다.
- 파싱된 번역 항목을 key, KR, EN으로 검색할 수 있다.
- 화면 이미지를 업로드할 수 있다.
- 화면 위에 텍스트 영역을 수동으로 지정할 수 있다.
- 텍스트 영역에 번역 항목을 연결할 수 있다.
- 오른쪽 리스트에서 연결된 문구의 11개 언어 번역본이 기본 노출된다.
- 화면 텍스트와 오른쪽 번역 항목이 양방향으로 하이라이트된다.
- 연결 상태와 메모가 저장된다.
- 페이지를 새로고침해도 저장된 화면, 텍스트 영역, 연결 정보가 유지된다.

## 6. 구현 우선순위

### Phase 1. 데이터 기반
- HTML 업로드
- HTML 파싱
- TranslationItem 저장
- 번역 항목 검색

### Phase 2. 화면 기반

### Phase 3. 연결 기반
- TextRegion과 TranslationItem 연결
- 오른쪽 다국어 번역 리스트 표시
- 상태 및 메모 저장

### Phase 4. 검수 편의
- 양방향 하이라이트
- 자동 스크롤
- 상태 필터
- 미연결 항목 표시

## 7. 판단 기준

MVP에서는 “자동으로 모든 것을 정확히 연결하는 것”보다, “사람이 확인한 연결 정보를 빠르게 축적하고 재사용할 수 있는 것”을 더 중요하게 본다.

따라서 아래 기능은 구현 품질 판단에서 우선순위가 높다.

- 텍스트 영역 지정이 빠른가
- 번역 항목 검색이 빠른가
- 11개 언어 번역본을 한 화면에서 확인하기 쉬운가
- 연결 상태와 메모를 남기기 쉬운가
- 화면 맥락과 번역 데이터를 함께 보기 쉬운가

## 화면 모드 구분

### View Mode: 기본 진입 화면

View Mode는 앱의 기본 화면이다.

포함 기능:
- 등록된 화면 선택
- 선택된 화면 이미지 조회
- 화면 위 텍스트 영역 overlay 조회
- 화면 텍스트 클릭 시 오른쪽 번역 항목 하이라이트
- 오른쪽 번역 항목 클릭 시 화면 텍스트 영역 하이라이트
- 화면별 다국어 번역 테이블 조회
- key, 화면 문구, 상태, 메모 조회
- 검색 및 상태 필터

View Mode에서는 화면 추가, 이미지 업로드, 텍스트 영역 생성/수정/삭제, key 연결/해제 기능을 직접 노출하지 않는다.

### Add/Edit Mode: 별도 진입

Add/Edit Mode는 “화면 추가” 또는 “편집 모드” 버튼을 눌렀을 때만 진입한다.

포함 기능:
- 화면 추가/수정
- 화면 이미지 업로드
- 텍스트 영역 생성/수정/삭제
- 번역 key 검색 및 연결/해제
- 상태 및 메모 수정
