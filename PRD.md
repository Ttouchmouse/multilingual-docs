# PRD: 화면 기반 다국어 텍스트 QA 맵

## 1. 제품 개요

### 제품명
가칭: Multilingual Text Map

### 목적
글로벌 성인 웹툰 서비스에서 사용 중인 다국어 UI 텍스트를, 단순 번역 리스트가 아니라 실제 화면 맥락 기준으로 조회·검수·관리할 수 있는 개인용 내부 도구를 만든다.

현재 다국어 번역본은 HTML 파일에 전체 리스트 형태로 관리되고 있으나, 특정 문구가 실제 서비스의 어느 화면, 어느 영역에 사용되는지 확인하기 어렵다. 본 도구는 피그마 기준 주요 화면 이미지를 등록하고, 화면 위 텍스트 영역과 DB화된 다국어 번역 항목을 연결하여, 화면 단위로 11개 언어 번역본을 확인할 수 있게 한다.

## 2. 배경 및 문제 정의

현재 운영 구조는 다음과 같다.

- 서비스는 KR, EN, SC, TC, ES, IT, PT, DE, FR, JP, TH 총 11개 언어권을 대상으로 한다.
- 다국어 번역본은 HTML 파일로 관리된다.
- 번역 요청은 별도 스프레드시트에 적재하여 담당 부서에 요청한다.
- 피그마에는 주요 화면 산출물이 존재하지만, 피그마 화면과 실제 다국어 번역 리스트는 직접 연결되어 있지 않다.

현재 문제는 다음과 같다.

1. HTML 파일은 전체 번역본 조회에는 가능하지만, 특정 문구가 실제 어느 화면에 사용되는지 알기 어렵다.
2. 화면에 보이는 문구와 HTML 내 번역 항목을 사람이 매번 수동으로 대조해야 한다.
3. 같은 문구가 여러 key에 존재할 경우 어떤 항목이 실제 해당 화면에 쓰이는지 판단하기 어렵다.
4. 다국어 번역본을 화면 맥락에서 검수하기 어렵다.
5. 한 번 확인한 사용처 정보가 체계적으로 축적되지 않아 동일한 확인 작업이 반복된다.

## 3. 제품 목표

MVP의 목표는 다음과 같다.

- 피그마 기준 주요 화면을 이미지로 등록한다.
- 화면 위 텍스트 영역을 툴 내에서 수동 지정한다.
- 각 텍스트 영역을 DB화된 다국어 번역 항목의 key와 연결한다.
- 화면 기준으로 해당 문구의 11개 언어 번역본을 기본 노출한다.
- 화면 내 텍스트와 오른쪽 번역 항목을 양방향으로 하이라이트한다.
- 연결 상태, 확인 필요 여부, 메모를 누적한다.

## 4. 핵심 사용자

### 1차 사용자
- 프로덕트 디자이너

### 보조 사용자
- 기획자
- 번역 요청 담당자
- QA 담당자
- 개발자

단, MVP는 개인용 도구로 시작한다. 다중 사용자 권한, 승인 플로우, 실시간 협업은 포함하지 않는다.

## 5. 핵심 사용 시나리오

### 시나리오 1. 화면 기준 다국어 번역 확인
사용자는 결제 화면 PNG를 등록한다. 화면 위의 CTA 텍스트 영역을 박스로 지정하고, 기존 HTML에서 파싱된 번역 항목 중 해당 문구와 맞는 key를 연결한다. 이후 화면에서 CTA를 클릭하면 오른쪽에서 KR, EN, SC, TC, ES, IT, PT, DE, FR, JP, TH 번역본을 바로 확인할 수 있다.

### 시나리오 2. 문구 사용처 누적
사용자는 업무 중 특정 문구가 실제 서비스 화면에 사용 중임을 확인한다. 해당 화면의 텍스트 박스에 상태를 “연결 완료”로 설정하고, 메모에 “결제 페이지 하단 CTA / DE 길이 주의”를 남긴다.

### 시나리오 3. 확인 필요 문구 관리
동일한 KR 문구가 여러 key에 존재해 정확한 연결이 불확실한 경우, 사용자는 임시로 가장 유력한 항목을 연결하되 상태를 “확인 필요”로 설정한다. 추후 실제 서비스 또는 개발자 확인 후 상태를 갱신한다.

## 6. 주요 기능

### 6.1 번역 HTML 업로드 및 파싱
- 기존 HTML 파일을 업로드한다.
- HTML 내 다국어 번역 테이블을 파싱한다.
- 각 행을 번역 항목으로 저장한다.
- 11개 언어 컬럼을 유지한다.
- key 또는 식별자가 존재하는 경우 문구 ID로 사용한다.
- key가 명확하지 않은 경우 내부 ID를 생성한다.

### 6.2 화면 추가
- 사용자는 피그마에서 export한 주요 화면 이미지를 업로드한다.
- 화면명, 화면 그룹, 플랫폼, 기준 언어, 피그마 링크, 메모를 입력할 수 있다.
- 화면 그룹 예: 결제, 구독, 성인인증, 쿠키, 로그인, 회원가입, 뷰어, 이벤트, 마이페이지.

### 6.3 텍스트 영역 지정
- 업로드된 화면 이미지 위에서 드래그하여 텍스트 영역 박스를 생성한다.
- 각 박스에는 화면에 보이는 텍스트를 입력할 수 있다.
- 박스는 이동, 크기 조정, 삭제가 가능해야 한다.
- 박스 선택 시 오른쪽의 연결된 번역 항목이 하이라이트된다.

### 6.4 번역 항목 연결
- 사용자는 화면 텍스트 박스에 DB화된 번역 항목 key를 연결한다.
- 연결 시 검색 기준은 KR, EN, key를 기본으로 한다.
- 동일 문구가 여러 개 존재할 수 있으므로, 검색 결과에서 사용자가 직접 선택한다.
- 연결 확신이 낮은 경우 상태를 “확인 필요”로 남길 수 있다.

### 6.5 화면 기준 11개 언어 번역본 기본 노출
- 오른쪽 영역에는 해당 화면에 연결된 모든 텍스트 항목이 나열된다.
- 각 항목은 11개 언어 번역본을 기본 노출한다.
- 각 항목에는 key, 화면 문구, 상태, 메모가 함께 표시된다.
- 화면 텍스트 클릭 시 해당 항목으로 자동 스크롤 및 하이라이트된다.
- 오른쪽 항목 클릭 시 화면 내 텍스트 영역이 하이라이트된다.

### 6.6 상태 및 메모 관리
각 텍스트 연결 항목은 다음 상태를 가질 수 있다.

- 미연결
- 연결 완료
- 확인 필요
- 번역 누락
- 수정 필요

메모에는 다음 내용을 기록할 수 있다.

- 실제 사용 위치
- UI 길이 리스크
- 실서비스와 피그마 문구 차이
- 동일 문구 중복 여부
- 번역 요청 필요 여부
- QA 확인 내용

## 7. 화면 구성

### 7.x View Mode 기본 원칙

본 도구의 기본 진입 화면은 등록/편집 화면이 아니라 조회 화면이다.

View Mode는 Figma에서 정리한 조회 모드 디자인을 기준으로 한다.

Figma 기준:
- 상단 헤더
- 좌측 화면 리스트
- 중앙 화면 프리뷰
- 우측 다국어 번역 테이블
- 화면 위 텍스트 overlay
- overlay와 번역 테이블 row 간 양방향 하이라이트

화면 추가, 이미지 업로드, 텍스트 영역 지정, 번역 key 연결 기능은 유지한다. 단, 해당 기능들은 기본 조회 화면에 직접 노출하지 않고, 화면 추가 또는 편집 버튼을 통해 접근한다.

### 7.1 메인 화면
- 좌측: 화면 이미지 뷰어
- 우측: 해당 화면에 사용된 텍스트별 11개 언어 번역 리스트
- 상단: 화면 선택, HTML 데이터 상태, 검색
- 하단 또는 사이드: 상태 필터

### 7.2 화면 추가 화면
- 화면명
- 화면 그룹
- 기준 언어
- 플랫폼
- Figma URL
- 화면 이미지 업로드
- 메모

### 7.3 번역 데이터 관리 화면
- 업로드된 HTML 파일 정보
- 파싱된 번역 항목 수
- 언어별 누락 수
- 전체 번역 항목 검색

## 8. 성공 기준

MVP 성공 기준은 다음과 같다.

- HTML 파일을 업로드하여 11개 언어 번역 데이터를 파싱할 수 있다.
- 최소 1개 이상의 주요 화면 이미지를 등록할 수 있다.
- 화면 위 텍스트 영역을 수동 지정할 수 있다.
- 텍스트 영역과 번역 key를 연결할 수 있다.
- 화면 기준으로 11개 언어 번역본을 기본 노출할 수 있다.
- 화면 텍스트와 번역 항목 간 양방향 하이라이트가 동작한다.
- 연결 상태와 메모를 저장할 수 있다.

## 9. 비목표

MVP에서는 다음을 하지 않는다.

- Figma API 자동 연동
- Figma text layer 자동 추출
- OCR 기반 텍스트 자동 인식
- 코드 저장소 기반 실제 i18n key 사용처 자동 추적
- AI 자동 번역
- 번역 요청 스프레드시트 자동 제출
- 다중 사용자 권한 관리
- 실서비스와 실시간 동기화
- 운영 서비스 배포용 번역 파일 생성

## 10. 향후 확장 가능성

MVP 이후 확장 가능성은 다음과 같다.

- Figma URL 기반 프레임 이미지 자동 가져오기
- Figma text layer 추출
- OCR 기반 화면 텍스트 후보 자동 감지
- 유사 문구 검색
- 언어별 길이 리스크 자동 표시
- 번역 요청 스프레드시트 export
- 화면별 QA 체크리스트
- 변경 이력 관리
- 주요 컴포넌트 단위 다국어 검수

## 기본 화면 원칙

본 도구의 기본 진입 화면은 등록/편집 화면이 아니라 조회 화면이다.

사용자는 기본적으로 이미 등록된 피그마 기준 화면 이미지와, 해당 화면에 연결된 다국어 번역 테이블을 확인한다.

화면 추가, 이미지 업로드, 텍스트 영역 지정, 번역 key 연결, 상태/메모 수정은 별도의 Add/Edit Mode에서 수행한다.

등록된 화면이 하나 이상 존재하는 경우, 앱 진입 시 가장 최근에 선택했거나 첫 번째 등록된 화면의 View Mode를 먼저 보여준다.

등록된 화면이 없는 경우에만 Empty State를 보여주며, 이때 “화면 추가” CTA를 제공한다.
