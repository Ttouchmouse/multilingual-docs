# DESIGN: TG 다국어 위키

## 1. 디자인 기준

본 문서는 TG 다국어 위키의 구현 기준 디자인 문서다.

기본 진입 화면은 등록/편집 화면이 아니라 조회 중심 View Mode다.  

등록된 화면이 하나 이상 존재하는 경우, 앱은 화면 추가 폼이 아니라 View Mode를 먼저 보여준다.

화면 추가, 이미지 업로드, 텍스트 영역 지정, 번역 key 연결 기능은 유지하되, 기본 조회 화면의 주 UI로 노출하지 않는다. 해당 기능은 별도 버튼 또는 편집 흐름을 통해 접근한다.

## 3. View Mode 레이아웃

View Mode는 다음 4개 영역으로 구성한다.

### 3.1 Header

- 화면 상단 고정 영역

- 좌측에 서비스명 `TG 다국어 위키` 표시

- 우측에 `편집` 버튼 표시

- 기본 View Mode에서는 HTML 업로드, 로컬 HTML 불러오기, 화면 등록 폼을 노출하지 않는다.

### 3.2 Left Sidebar

- 등록된 주요 화면 리스트를 표시한다.

- 사용자는 사이드바에서 화면을 선택할 수 있다.

- 선택된 화면은 active 상태로 표시한다.

- 예시 메뉴:

  - Home

  - 결제 페이지

  - 쿠키 팝업

  - 성인인증

  - 로그인/회원가입

### 3.3 Center Preview

- 선택된 화면 이미지를 표시한다.

- 피그마에서 export한 화면 이미지를 기준으로 보여준다.

- 화면 위에는 연결된 텍스트 영역 overlay를 표시한다.

- overlay는 클릭 가능해야 한다.

- overlay 클릭 시 우측 번역 테이블의 해당 row가 하이라이트되고, 필요 시 해당 row로 자동 스크롤된다.

- 선택된 overlay는 일반 overlay와 명확히 구분되어야 한다.

### 3.4 Right Translation Table

- 선택된 화면에 연결된 텍스트별 다국어 번역 테이블을 표시한다.

- 각 row는 화면 내 텍스트 영역 1개에 대응한다.

- 각 column은 언어 1개에 대응한다.

- 언어 순서는 다음을 기본으로 한다.

`KR / EN / SC / TC / ES / IT / PT / DE / FR / JP / TH`

- row 클릭 시 중앙 화면 프리뷰의 해당 overlay가 하이라이트된다.

- 선택된 row는 일반 row와 명확히 구분되어야 한다.

## 4. Empty State

등록된 화면이 없는 경우에만 Empty State를 표시한다.

Empty State 문구 예시:

`등록된 화면이 없습니다. 화면 추가를 통해 첫 화면을 등록하세요.`

등록된 화면이 하나 이상 존재하는 경우에는 Empty State나 화면 추가 폼이 아니라 View Mode를 먼저 표시한다.

## 5. Add/Edit 기능 처리

기존 화면 추가, 이미지 업로드, 텍스트 영역 지정, 번역 key 연결 기능은 삭제하지 않는다.

다만 해당 기능은 기본 View Mode에 직접 노출하지 않는다.

접근 방식:

- Header 또는 View Mode 내 `편집` 버튼을 통해 진입

- 또는 `화면 추가` 버튼을 통해 별도 추가 플로우로 진입

이번 작업의 핵심은 기능 삭제가 아니라 View Mode 레이아웃 리팩터링이다.

## 6. 구현 시 주의사항

- 기존 데이터 구조는 최대한 유지한다.

- 기존 화면 추가/편집 기능은 삭제하지 않는다.

- 기본 화면을 등록 폼 중심으로 만들지 않는다.

- 등록된 화면이 있으면 View Mode를 우선 표시한다.

- Figma URL, 플랫폼 선택, HTML 수동 업로드 UI는 기본 View Mode에 노출하지 않는다.

- 번역 데이터는 앱 실행 시 기본 데이터로 자동 로드된 상태를 전제로 한다.
